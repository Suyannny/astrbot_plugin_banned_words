# 违禁词管理插件
AI写的，有能力的自行修改，随缘更新……
## 插件简介

**违禁词管理插件** 是一款适配 AstrBot 4.20.0 的机器人功能管理工具，旨在帮助主人有效管理机器人群聊内容，维护良好的聊天环境。该插件提供了完整的违禁词管理功能，包括添加、删除、列出和清空违禁词，并能自动拦截包含该违禁词的机器人指令功能，做到机器人无视该指令。

### 核心特性

- **违禁词管理**：支持添加、删除、列出、清空违禁词，操作简单直观
- **消息自动拦截**：实时监测群聊消息，自动拦截包含违禁词的内容
- **权限分级控制**：主人和管理员权限分离，确保管理安全
- **管理员授权系统**：主人可授权普通用户成为特定群聊的管理员
- **数据持久化**：所有配置和数据自动保存，重启后自动恢复

## 安装方法

### 方法二：手动安装

1. 将插件文件夹 `astrbot_plugin_banned_words` 复制到 AstrBot 的 `data/plugins/` 目录下
2. 重启 AstrBot 或在管理面板中刷新插件列表
3. 在插件列表中启用「违禁词管理」插件

## 工作流程

### 1. 插件初始化流程

```
插件加载
    ↓
读取配置文件（_conf_schema.json）
    ↓
加载主人列表（master_users）
    ↓
加载管理员配置（admin_users）
    ↓
加载违禁词配置（banned_words）
    ↓
读取运行时数据文件（banned_words_data.json）
    ↓
合并配置文件和运行时数据
    ↓
插件初始化完成
```

### 2. 消息拦截流程

```
群消息到达
    ↓
检查发送者身份
    ├── 是主人 → 拦截，也可设置放行
    ├── 是管理员 → 拦截，也可设置放行
    └── 普通用户 → 拦截
    ↓
检查消息是否包含违禁词
    ├── 包含违禁词 → 拦截该违禁词触发的功能
    └── 不包含 → 放行
```

### 3. 权限验证流程

```
用户执行指令
    ↓
检查指令类型
    ├── 违禁词管理指令 → 检查是否为主人或管理员
    ├── 管理员授权指令 → 检查是否为主人
    ├── 主人管理指令 → 检查是否为主人
    └── 帮助/列表指令 → 无需权限检查
    ↓
权限验证结果
    ├── 通过 → 执行指令
    └── 不通过 → 返回权限不足提示
```

### 4. 数据持久化流程

```
数据变更操作
    ↓
修改内存中的数据结构
    ↓
调用 _save_data() 方法
    ↓
将数据写入 banned_words_data.json
    ↓
数据持久化完成
```

## 权限说明

### 权限等级

| 等级 | 角色 | 权限范围 |
|------|------|----------|
| 1 | 主人 | 拥有全部权限，可管理所有群聊的违禁词，可授权/取消管理员，可添加/删除主人 |
| 2 | 管理员 | 只能管理被授权群聊的违禁词，无法授权其他用户 |
| 3 | 普通用户 | 只能查看违禁词列表和管理员列表，消息会被违禁词过滤 |

### 权限继承关系

- **主人默认为所有群聊的管理员**：主人无需额外授权即可在任何群聊中执行管理员操作
- **管理员权限仅限授权群聊**：管理员只能在被授权的群聊中执行违禁词管理操作
- **主人和管理员的消息不被拦截**：即使消息包含违禁词，主人和管理员发送的消息也不会被拦截

## 注意事项

1. **首次使用**：请务必阅读常见问题一览，在FORCE_MASTER_ID中设置主人QQ号，否则无法使用相关功能
2. **主人保护**：系统至少保留一个主人，无法删除最后一个主人
3. **数据安全**：所有数据会自动保存到 `banned_words_data.json` 文件中
4. **配置优先级**：配置文件中的设置会与运行时数据合并，不会覆盖
5. **大小写不敏感**：违禁词匹配不区分大小写

## 常见问题

### Q1: 如何设置第一个主人？

**A:** 打开webui中的插件配置，强制设定的最高权限主人QQ号设置为您的的QQ号。

### Q2: 管理员和主人有什么区别？

**A:** 主人拥有全部权限，可以管理所有群聊的违禁词，还可以授权其他用户为管理员。管理员只能在被授权的群聊中管理违禁词，无法授权其他用户。

### Q3: 为什么我的消息被拦截了？

**A:** 如果您不是主人或管理员，发送包含违禁词的消息会被自动拦截。请联系主人或管理员将您添加为管理员，或者从违禁词列表中移除相关词汇。

### Q4: 为什么主人和管理员的消息也被拦截了？

**A:** 打开webui中的插件配置，主人和管理员是否豁免违禁词检查这项开启就不会拦截。

### Q5: 为什么拦截后没有任何提示？

**A:** 打开webui中的插件配置，拦截违禁词时是否发送提示消息这项开启就会提示。

### Q6: 为什么指令没有反应？

**A:** 自行检查前缀或指令问题

### Q7: 数据会丢失吗？

**A:** 不会。所有运行时数据都会自动保存到 `banned_words_data.json` 文件中，重启插件后会自动加载。
